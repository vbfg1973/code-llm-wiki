# Plans Structure Standard

Backlog tracker:

- `/plans/BACKLOG.md`

Agent orientation docs:

- `/AGENTS.md`
- `/CLAUDE.md`
- `/CODEX.md`

Directory and file naming standard:

- `/plans/{prd_id}/{prd_id}-{name}.prd.md`
- `/plans/{prd_id}/{prd_id}-{name}.plan.md`
- `/plans/{prd_id}/{prd_id}-{name}.issue.{issue_id}.md`

ID rules:

- `prd_id` and `issue_id` are left-padded to 3 digits (for example `001`, `024`).

Execution checklist standard (going forward):

- Plan elements should be tracked via checklists.
- As each item is completed, mark it checked and record completion date inline.
- Existing historical plans are not required to be retrofitted for checklist formatting.
